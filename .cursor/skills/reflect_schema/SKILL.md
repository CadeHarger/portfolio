## When to Use

- Use this skill only when invoked by the user

## Instructions
Reflect upon the yaml documentation that you used in this thread. Were the parts you payed most attention to helpful? What could've been mentioned that wasn't, and that would've saved a lot of tokens? Can the related part of the documentation be more concise?

Reflect upon this and then update the documentation accordingly. Beware of documentation bloat and only mention what is necessary. If we mention every little detail, it might be good for changes involving the feature, but in 90% of cases it just wastes tokens.
